Areas de Git

working directory

es la carpeta en la que iniciamos el repositorio, donde realizamos los cambios en los archivos y rara vez estaremos utilizando la consola.

  • mkdir <dir>;DIR
  • cd <dir>
  • cd <dir>
  • git init

    • staging Area (INDEX)

    Es el area de control de cambios, un archivo que identifica y verifica los cambios realizados en el directorio de trabajo. Todo archivo al que querramos hacerle seguimiento, estar agregado previamente al Index para que se realice su snapshot

    • git status compara el Working Directory con el index y notifica las diferencias que existen entre ambos. Este comando notifica sobre el estado de los archivos.
      • untrackedarchivo que no se encuentra en el Index
      • trackedarchivo al que se le esta haciendo seguimiento
      • modifiedarchivo que difiere con lo registrado en el index
      • commitedarchivo cuyos cambios ya fueron confirmados
    • git add <file>agrega al index los archivos seleccionados, podemos acelerar esta tarea utilizando la expresion gitt add . pero tenemos que tener en cuenta que esto agrega todos los cambios del directorio actual.
    • git rm --cached <flie>remueve el archivo seleccionado del index, pero mantiene los cambios en el WORKING DIRECTOY. Esto funciona en el caso que hayamos agregado el archivo al index y todavia no exista en el historial.
    • git restore <flie>restablece el estado del archivo seleccionado a como se encontraba en el INDEX, antes de realizar los cambios.

    Repositoriy (Local/Remote)

    Es el area donde se almacenan los cambios confirmados, en formato BLOB (binare object File. esto nos permite restablecer tanto archivos, directorios, o hasta el mismo proyecto utilizando como referencia dichos archivos e indicando a git en que instancia del mismo.
